## 1. 배경 (Why)

홈은 온보딩 완료 후 사용자가 매일 처음 만나는 **일일 대시보드**다. 육아밸의 핵심 가치인 "아이 정보 기반 맞춤 답변·기록"을 한 화면에서 시작할 수 있어야 한다.

`Home_v04`는 다음 정보를 한 화면에 묶는다.

- 주간 캘린더와 요일별 마음/기분 체크 상태
- 오늘의 미션 CTA
- 현재 자녀 발달 단계 요약
- 이번 달 함께한 날 비율
- 주요 탭(홈, 10분 놀이, 성장 로드맵, AI 상담, 주간 리포트)

`Home_v03`는 다자녀 사용자가 상단 자녀 피커를 눌렀을 때 자녀 목록을 열고, 선택·수정·삭제할 수 있는 상태를 보여준다. 따라서 홈은 **선택된 자녀 기준으로 모든 카드 데이터가 바뀌는 구조**여야 한다.

---

## 2. 사용자 시나리오 (What)

- **누가**: 온보딩을 완료한 부모 사용자. 다자녀 사용자를 포함한다.
- **언제**: 앱/WebView 진입 후 `/` 홈으로 도착할 때. 또는 하단 탭 "홈"을 누를 때.
- **흐름**:
  1. 사용자는 홈 상단에서 현재 선택된 자녀(`김유스 (만3세)`)와 알림 아이콘을 본다.
  2. 사용자는 주간 캘린더에서 현재 주차(`4월`, `3주차`)와 오늘(`수 15`)을 확인한다.
  3. 사용자는 요일별 마음/기분 체크 버튼을 확인하거나 오늘 체크로 진입한다.
  4. 사용자는 오늘의 미션 카드에서 추천 미션을 보고 "시작하기"를 누른다.
  5. 사용자는 현재 상황 카드에서 자녀의 발달 단계(`자아 형성기`)와 짧은 설명을 본다.
  6. 사용자는 리포트 카드에서 이번 달 함께한 날 비율(`84%`)을 확인하고 주간 리포트 탭으로 이동할 수 있다.
  7. 사용자는 알림 아이콘을 눌러 홈 위에 알림 목록 모달을 연다.
  8. 사용자는 상단 자녀 피커를 눌러 자녀 목록을 연다. 자녀가 1명이어도 목록은 열린다.
  9. 목록에서 "수정"은 자녀 정보 수정 화면으로 이동하고, "삭제"는 확인 다이얼로그 후 soft delete를 요청한다.
- **수용 기준**:
  - 홈 데이터는 항상 **선택된 자녀 기준**으로 조회·표시된다.
  - 온보딩 미완료 사용자는 홈에 접근하지 못하고 온보딩으로 리디렉션된다.
  - 자녀 수와 무관하게 자녀 피커는 목록 오버레이를 연다. 1명뿐이면 한 명만 표시한다.
  - 미션 CTA는 미션 상세/타이머 시작 플로우로 이동한다.
  - 알림 아이콘은 페이지 이동 없이 홈 위에 알림 목록 모달을 연다.
  - 하단 탭은 한국어 `Home_v04` 라벨을 기준으로 한다.

## 3. 도메인 영향

홈 자체는 새 도메인 테이블보다 **기존 도메인의 집계/조회 계약**이 핵심이다.

| 엔티티                         | 변경 종류                              | 비고                                                                  |
| ------------------------------ | -------------------------------------- | --------------------------------------------------------------------- |
| `Child`                        | 필드 추가 — `deletedAt: DateTime?`     | 홈 선택 단위. 삭제는 soft delete로 처리. 조회 시 `deletedAt == null`  |
| `GrowthStage`                  | 변경 없음                              | 현재 상황 카드의 `자아 형성기`, 요약 문구                             |
| `Mission`                      | 변경 없음                              | 오늘 추천 미션 카드. `title`, `subThemeLabel`, `durationMinutes` 사용 |
| `MissionExecution`             | 변경 없음                              | 요일별 수행 상태, 이번 달 함께한 날 비율 산출                         |
| `MentalBatteryCheck`           | 변경 없음                              | 요일별 마음 체크 상태 표시                                            |
| `HomeDashboard` (DTO, DB 아님) | **신규 응답 DTO**                      | 여러 도메인을 조합한 API 응답                                         |
| `ChildProfileMutation`         | 기존/신규 endpoint 필요 여부 확인 필요 | 오버레이의 수정/삭제 액션                                             |
| `Notification`                 | 신규 schema 작성                       | 알림 아이콘 클릭 시 홈 모달에 표시                                    |

> `Child.deletedAt` soft delete 결정은 [`../schema/02-child.md`](../schema/02-child.md)에도 반영 필요. 알림 도메인은 [`../schema/11-notification.md`](../schema/11-notification.md) 기준으로 구현한다.

### 4.1 yougabell-api 상세 스펙

#### Prisma schema diff

```prisma
model Child {
  // ... 기존 필드 유지
  deletedAt DateTime? // nullable. 홈 자녀 삭제는 soft delete
}
```

- 마이그레이션: `pnpm prisma:migrate:dev --name child_soft_delete`
- `GET /home`을 포함한 사용자 화면 조회는 기본적으로 `Child.deletedAt == null` 조건을 적용한다.

#### 엔드포인트 — `GET /home`

**인증**: `Authorization: Bearer <Supabase JWT>` 필수. `JwtAuthGuard` + `OnboardingCompleteGuard` 적용.

**Query**:

```typescript
type GetHomeQuery = {
  childId?: string; // 없으면 displayOrder 가장 앞 자녀 또는 최근 선택 자녀
  date?: string; // YYYY-MM-DD, 기본 today. 테스트/프리뷰용
};
```

**처리 로직**:

1. `userId`의 자녀 목록을 `displayOrder ASC`로 조회
2. `childId`가 있으면 해당 자녀가 사용자 소유이고 `deletedAt == null`인지 검증
3. `childId`가 없으면 기본 자녀 선택
4. 선택 자녀의 월령 계산
5. 현재 주(월~일) 범위 계산
6. `MentalBatteryCheck`를 주간 범위로 조회해 요일별 `mood` 매핑
7. `MissionExecution`을 주간/월간 범위로 조회해 수행 상태와 이번 달 함께한 날 비율 계산
8. `GrowthStage` 매칭으로 현재 상황 카드 구성
9. `Mission` 추천 로직으로 오늘 미션 1건 구성
10. 알림 미확인 개수와 최신 알림 목록 일부를 포함
11. `HomeDashboard` 반환

#### 리포트 카드 산식 — 이번 달 함께한 날

`84%`는 `MissionExecution` 기반 **이번 달 함께한 날 비율**이다.

```text
이번 달 함께한 날 비율 =
  이번 달 1일~오늘 중 completed/early_completed 미션이 1개 이상 있는 날짜 수
  /
  이번 달 1일~오늘까지 지난 날짜 수
```

- 분자: 선택 자녀의 `MissionExecution.status IN ('completed', 'early_completed')`이고 `completedAt`이 이번 달 1일 00:00부터 오늘 23:59:59 사이인 기록을 날짜 단위로 중복 제거한 개수
- 분모: 이번 달 1일부터 오늘까지의 날짜 수 (`today.dayOfMonth`)
- 같은 날 미션을 여러 개 완료해도 해당 날짜는 1일로만 계산
- 날짜 경계는 사용자 기준 타임존(`Asia/Seoul` 우선, 추후 사용자 타임존 필드 생기면 해당 값)을 따른다.
- 예: 5월 11일 기준 완료한 날짜가 9일이면 `9 / 11 = 81.8%` → `82%`

**Response 200**:

```typescript
type GetHomeResponse = HomeDashboard;
```

**에러 응답**:

| 상태 | code                  | 의미                                    |
| ---- | --------------------- | --------------------------------------- |
| 400  | `VALIDATION_ERROR`    | `childId`, `date` 형식 오류             |
| 401  | `UNAUTHORIZED`        | JWT 없음/만료                           |
| 403  | `ONBOARDING_REQUIRED` | 온보딩 미완료                           |
| 404  | `CHILD_NOT_FOUND`     | 자녀 없음 또는 사용자 소유 자녀가 아님  |
| 409  | `NO_CHILD_PROFILE`    | 온보딩 완료 상태이나 자녀 데이터가 없음 |
| 500  | `INTERNAL_ERROR`      | 집계 실패                               |

#### 엔드포인트 — `PATCH /children/:id` (존재 여부 확인)

홈 오버레이의 "수정"은 별도 자녀 수정 화면으로 이동하는 것이 우선이다. API 계약은 온보딩 자녀 필드와 동일하게 최소 수정 가능해야 한다.

```typescript
type UpdateChildDto = {
  name?: string;
  birthDate?: string; // YYYY-MM-DD
  gender?: "female" | "male";
  notes?: string | null;
  displayOrder?: number;
};
```

#### 엔드포인트 — `DELETE /children/:id`

삭제는 soft delete로 처리한다.

- 자녀가 1명뿐이면 삭제 차단: `409 LAST_CHILD_CANNOT_BE_DELETED`
- 연결된 `MissionExecution` 등 자녀 기준 기록은 보존
- 처리: `Child.deletedAt = now()`
- `displayOrder` 재정렬은 남은 active 자녀 기준으로 후속 처리
- 이미 삭제된 자녀는 `404 CHILD_NOT_FOUND` 또는 멱등 `204` 중 하나로 확정 필요

#### 엔드포인트 — 알림 목록/읽음

알림 아이콘 클릭 시 페이지 이동 없이 홈 위에 알림 목록 모달을 연다.

구현 옵션:

- `GET /home`에 `notifications.unreadCount`와 최신 N개를 embedded
- 또는 `GET /notifications?limit=20`을 모달 open 시 호출

초기 구현은 홈 첫 렌더의 벨 badge를 위해 `GET /home.notifications.unreadCount`를 포함하고, 모달 open 시 `GET /notifications`를 호출하는 방식을 권장한다.

**Target 계약**은 [`../schema/11-notification.md`](../schema/11-notification.md) 기준이다. JSON payload를 두지 않고 `actionType`, `targetType`, `targetId`, `targetUrl`로 클릭 목적지를 표현한다.

읽음 처리는 [`../schema/11-notification.md`](../schema/11-notification.md) 기준이다.

- 모달 open/목록 노출만으로는 읽음 처리하지 않는다.
- 개별 알림 클릭 시 해당 알림을 읽음 처리한다.
- 알림 모달에는 "모두 읽음" 액션을 제공한다.
- 알림 보존 기간은 무제한으로 시작한다.

**엔드포인트 계약**:

| Method  | Path                      | 설명                                     |
| ------- | ------------------------- | ---------------------------------------- |
| `GET`   | `/notifications?limit=20` | 최신 알림 목록 조회                      |
| `PATCH` | `/notifications/:id/read` | 개별 알림 클릭 시 읽음 처리              |
| `PATCH` | `/notifications/read-all` | 현재 사용자의 미확인 알림 모두 읽음 처리 |

#### OpenAPI export

- `GET /home` 응답 DTO를 OpenAPI에 포함
- 자녀 수정/삭제 endpoint가 새로 생기면 web/mobile codegen 갱신

#### 상태 스키마

선택 자녀는 서버 저장 없이 클라이언트 로컬 선호값으로 시작한다.

```typescript
type HomeClientState = {
  selectedChildId: string | null;
  isChildSwitcherOpen: boolean;
  isNotificationModalOpen: boolean;

```

- localStorage 키 후보: `home:selected-child-id`
- 서버 `GET /home?childId=` 응답이 `404 CHILD_NOT_FOUND`이면 키 삭제 후 기본 자녀 재조회
- 멀티 디바이스 동기화는 미지원. 홈 편의 상태라 손실 허용

### 4.3 yougabell-mobile 상세 스펙

홈 UI는 web이 담당한다. mobile은 기존 WebView 컨테이너가 다음을 만족하면 추가 구현 없음.

| 항목           | 책임   | 요구 사항                                                             |
| -------------- | ------ | --------------------------------------------------------------------- |
| Safe area      | web    | 하단 탭이 iOS home indicator와 겹치지 않도록 `safe-area-inset-bottom` |
| Android back   | mobile | 자녀 오버레이 열린 상태에서 web history 또는 in-page state 닫기 우선  |
| Push 알림      | mobile | OS 권한/푸시 토큰만 담당. 홈 알림 목록 모달은 web이 담당              |
| WebView cookie | mobile | Supabase 세션 유지                                                    |

### 4.4 엣지 케이스

## 6. 비기능 요구

### 저장 전략

**서버 조회 + 선택 자녀만 디바이스 로컬 저장.**

```
[홈 진입]                         [web]                         [api]
localStorage selectedChildId ──→ GET /home?childId=... ──────→ 여러 도메인 집계
                                      ↓ 200
                                  HomeDashboard render

[자녀 선택]                       localStorage 갱신
                                  GET /home?childId=next
```

- **서버 DB**: 홈에 필요한 원천 데이터는 기존 도메인 테이블에 저장
- **로컬 저장**: `home:selected-child-id`만 저장
- **단계별 저장**: 없음
- **영속성 등급**: 앱 삭제·브라우저 캐시 삭제 시 선택 자녀 초기화 수용
- **멀티 디바이스**: 선택 자녀 동기화 미지원
- **캐시**: 홈 응답은 짧은 TTL(예: 30~60초) 또는 route revalidation 검토. 미션/마음 체크 직후에는 강제 refetch

### 그 외

- **성능**: 홈 첫 로딩에 필요한 데이터는 `GET /home` 1회 호출로 받는다. web에서 카드별로 여러 endpoint를 연속 호출하지 않는다.
- **보안**: 모든 홈 데이터는 사용자 소유 자녀 기준으로 필터링. `childId` 소유권 검증 필수.
- **접근성**:
  - 자녀 피커는 button + `aria-expanded`
  - 오버레이는 dialog/sheet role, ESC/배경 탭 닫기
  - 하단 탭은 현재 탭 `aria-current="page"`
  - 이모지 마음 버튼은 텍스트 대체 라벨 제공
- **분석 이벤트**:
  - `home_view`
  - `home_child_switcher_open`
  - `home_child_selected`
  - `home_mood_check_click`
  - `home_mission_start_click`
  - `home_notifications_open`
  - `home_report_card_click`
  - `home_bottom_nav_click`
